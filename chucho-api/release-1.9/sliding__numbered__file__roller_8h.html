Detailed Description

A file_roller that numbers files from a minimum index in increasing order.

New files have higher numbers than old files, and files are never renamed. This allows expensive rename operations on large sets of files to be avoided. You provide a maximum count, which is the total set of files, not the set of "rolled" ones. For example, if you have a rolling_file_writer with a file name of my.log, and your sliding_numbered_file_roller has a maximum count of 3, then the files will roll like this:

my.log
my.log.1
my.log.2

Now, on the next roll operation you will have the following set of files:

my.log.1
my.log.2
my.log.3

And the current active file is the one with the highest index.
