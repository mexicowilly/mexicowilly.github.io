Detailed Description

Stream to a log.

This stream can be used to write to a given log. The I/O manipulators trace, debug, info, warn, error, fatal and chucho::set_level, or the method log_stream::set_level can used to set the level of the stream. The stream's level is stored even after the log message is written. So, if there are four messages in a row at level::INFO_ level, then the level may just be set for the first. Additionally, a marker may be added to the current message using the I/O manipulator set_marker.

The macro CHUCHO_M should be used to start a log message. This macro does nothing more than set the location information of the current message into the stream, so if the location is not important, the macro can be omitted.

The I/O manipulator endm must be used to flush the message to the logger.

Typical usage is as follows.

chucho::log_stream stream(my_logger);
CHUCHO_M(stream) << chucho::info << "My dog has fleas." << chucho::endl << chucho::endm;

Bear in mind that the performance of streaming to a log can be worse than the performance using the log macros, such as CHUCHO_INFO. The reason is that when using the log macros, the arguments are are only evaluated if the message can actually be logged, given its level. When using the stream API, the arguments to build the message are always evaluated.

Construct a log stream.

All messages will be sent to the given logger. If no level is provided in the constructor, then the level must be set using log_stream::set_level or one of the manipulators trace, debug, info, warn, error, fatal or chucho::set_level. If endm is used to flush a message before any level has been set, then the message is dropped.
