Detailed Description

Types and functions common to all filters.

Conditionally filter an event. A writer may have a set of filters that provide finer-grained control of log output. Filter evaluation results in one of three values DENY, NEUTRAL, or ACCEPT. If the evaluation of all of a writer's filters results in NEUTRAL or ACCEPT, then the event is allowed to be written.

Enumeration Type Documentation

enum chucho_filter_result

The result of a filter operation.

Enumerator
CHUCHO_FILTER_RESULT_DENY 

The filter does not accept the event.

No subsequent filters are checked, as this is a vetoing result.

CHUCHO_FILTER_RESULT_NEUTRAL 

This filter doesn't care about the event.

Other filters in the chain are checked.

CHUCHO_FILTER_RESULT_ACCEPT 

The filter unconditionally accepts the event.

No subsequent filters are checked, as this is a vetoing result.
