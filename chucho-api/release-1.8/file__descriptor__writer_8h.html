Detailed Description

Writing to a file descriptor.

This is primarily used as a subclass for other writers, like file_writer. It is not usable in a configuration file because the file descriptor that would be used would have to be known before the application would launch in that context.

This class uses a buffer of 8 KB to cache outgoing messages. In the constructor of this class and in those of all subclasses this buffering can be enabled or disabled. By default, it is disabled. When buffering is enabled, the buffer is flushed only when it fills or when the writer is destroyed. Otherwise, it is flushed after every event is written.
