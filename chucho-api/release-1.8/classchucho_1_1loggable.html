Detailed Description

template<typename type>
class chucho::loggable< type >

A class that inherits from loggable gets a logger associated with each instance for free.

This gives you the ability to use the log macros tagged with LGBL in order to avoid having to manage the logger and what it might be called. For example,

CHUCHO_INFO_LGBL("I just found " << num << " cats");

Instead of,

CHUCHO_INFO(where_is_my_logger(), "I just found " << num << " cats");

By default the logger is named for the class (type template argument). If your class is called my::cool::stuff, then your logger will be named my.cool.stuff.

void chucho::loggable< type >::rename_logger ( const std::type_info &  new_type)
protected

Rename the logger.

This method is meant to be used in cases where a subclass of a subclass of loggable needs a different logger name. Say you have a class hierarchy built off the abstract class shape. You make shape a subclass of loggable, so that everyone gets a logger. Subclasses of shape don't want their loggers to be named shape, so they rename the logger in the subclass' constructor.
