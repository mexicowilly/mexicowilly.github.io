Detailed Description

Functions for creating and querying email writers.

A writer that sends log events via email. The writer holds events in a cache until its email_trigger indicates that it is time to send them. By default the cache holds 256 log events. When it is time to send email the log events are formatted into the email message body oldest to newest, one line at a time.

Chucho uses libcurl for implementation of the SMTP protocol. When it comes to SSL-based implementations, the libcurl that is present at run time must have SSL support. If it doesn't, then neither the connection_type::SSL nor the connection_type::STARTTLS connection types will be supported. Support for SSL can be detected before an email_writer is created with the static method get_ssl_supported.

enum chucho_email_connection_type

The type of connection that will be attempted with the email server.

Enumerator
CHUCHO_EMAIL_CONNECTION_CLEAR 

No encryption will be used.

CHUCHO_EMAIL_CONNECTION_SSL 

A pure SSL connection will be used.

Note
This connection type is only supported if get_ssl_supported returns true.
CHUCHO_EMAIL_CONNECTION_STARTTLS 

A STARTTLS connection will be used.

Note
This connection type is only supported if get_ssl_supported returns true.
