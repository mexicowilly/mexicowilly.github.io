Detailed Description

Conditionally filter an event.

A writer may have a set of filters that provide finer-grained control of log output. Filter evaluation results in one of three values DENY, NEUTRAL, or ACCEPT. If the evaluation of all of a writer's filters results in NEUTRAL or ACCEPT, then the event is allowed to be written.

◆ result

enum chucho::filter::result
strong

The result of filter evaluation.

Enumerator
DENY 

The event is denied.

This value prevents subsequent filters from being evaluated. It is a vetoing result.

NEUTRAL 

The event may pass.

Subsequent filters are evaluated, whose results may affect the ultimate outcome.

ACCEPT 

The event is accepted.

No other filters will be evaluated. The acceptance is unconditional.
