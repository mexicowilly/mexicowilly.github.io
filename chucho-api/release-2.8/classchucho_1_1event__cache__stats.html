Detailed Description

Information about event caches.

An event cache is a disk-backed queue of log events, which is used by event_cache_provider. The cache will report information about its state via the stats.

When the cache is full, a chunk of the oldest events is deleted. This can be monitored using a progress callback. The callback is invoked when the cache is 80%, 90% and 95% full.

See also
event_cache_provider

◆ progress_callback

using chucho::event_cache_stats::progress_callback = std::function<void(const event_cache_stats& stats, progress_direction dir, double full, std::size_t bytes_culled)>

Function used for progress callbacks.

If a callback has been set, it will be called when the cache is at 80%, 90% and 95% capacity. Additionally, it will be called after bytes have been culled, which occurs when the cache is full.

Parameters
statsthe stats at the time
dirwhether the cache is growing or shrinking
fullhow full the cache is in the range [0, 1]
bytes_culledthe number of bytes that were culled immediately before the callback was invoked
